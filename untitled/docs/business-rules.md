# قواعد العمل

## 1. قاعدة توفر Supabase

إذا لم تكن متغيرات البيئة التالية موجودة وصحيحة:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

فإن التطبيق يجب أن يبقى مستقراً ولا ينهار عند التحميل.

## 2. قاعدة التهيئة الكسولة

لا يجب إنشاء Supabase client أثناء تحميل الموديول.

يجب إنشاء العميل فقط عند استدعاء عملية تحتاج Supabase فعلياً، مثل:

- تسجيل الدخول
- إنشاء حساب
- قراءة الجلسة
- قراءة أو تحديث الملف الشخصي
- استدعاء Edge Function
- استعادة كلمة المرور
- حذف الحساب

## 3. قاعدة الخطأ

يجب عدم رمي الخطأ عند import أو عند بدء تشغيل الصفحة.

يجب أن يظهر الخطأ فقط عند محاولة تنفيذ عملية تحتاج Supabase.

رسالة الخطأ المعتمدة:

```text
Missing Supabase environment variables. Please set VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY in .env.local
```

## 4. قاعدة واجهة المستخدم

عند نقص البيئة:

- يجب عرض رسالة واضحة للمستخدم.
- يجب تعطيل الإجراءات التي لا يمكن تنفيذها.
- لا يجوز عرض نجاح وهمي.
- لا يجوز اعتبار العملية ناجحة إذا لم يتم تنفيذها فعلياً.

## 5. قاعدة المصادقة

إذا لم تكن Supabase مهيأة:

- لا يتم قراءة الجلسة.
- لا يتم الاشتراك في `onAuthStateChange`.
- لا يتم توجيه المستخدم إلى لوحة التحكم.
- لا يتم السماح بالدخول إلى المسارات المحمية.
- يجب أن يبقى المستخدم في صفحة عامة أو صفحة تسجيل دخول مع رسالة التهيئة الناقصة.

## 6. قاعدة الملف الشخصي

إذا لم تكن Supabase مهيأة:

- لا يتم إنشاء أو تحديث profile.
- لا يتم اعتبار profile موجوداً.
- أي محاولة لتحديث الملف الشخصي يجب أن ترجع رسالة خطأ واضحة.

## 7. قاعدة الإشعارات

إذا لم تكن Supabase مهيأة:

- لا يتم تحميل الإشعارات من قاعدة البيانات.
- يجب عرض حالة فارغة أو رسالة تعذر تحميل واضحة.
- لا يجوز عرض إشعارات وهمية.

## 8. قاعدة الحذف

حذف الحساب عملية حساسة ولا يجوز تنفيذها إلا إذا كان Supabase مهيأ بالكامل.

عند نقص البيئة:

- يتم تعطيل زر الحذف أو إظهار رسالة خطأ.
- لا يتم استدعاء Edge Function.
- لا يتم تغيير حالة المستخدم إلى محذوف.

## 9. قاعدة الاختبار

أي تغيير في منطق Supabase يجب أن يرافقه اختبار يثبت:

- استيراد ملف `src/lib/supabase.ts` لا يسبب crash عند نقص البيئة.
- `hasSupabaseEnv()` ترجع `false` عند نقص المتغيرات.
- العملية التي تحتاج Supabase لا تنجح بصمت عند نقص البيئة.

## 10. قاعدة عدم تغيير البنية

هذه المهمة لا تتطلب:

- migration جديدة.
- تغيير جداول Supabase.
- تغيير سياسات RLS.
- تغيير مزود المصادقة.

الهدف الوحيد هو تثبيت سلوك التطبيق عند نقص إعدادات Supabase.
